# Task Brief Template

Use this template when handing a feature or fix to an AI coding agent.

## Goal

What user-visible outcome should change?

## Scope

Routes, screens, components, or files likely affected.

## Constraints

Design, platform, performance, accessibility, or library constraints.

## API Contract

Endpoints, request payloads, response shapes, auth behavior, and error cases.

## Acceptance Criteria

- The behavior that must work
- The states that must be covered
- The checks that should pass

## Notes

Relevant decisions, known tradeoffs, or follow-up work.
